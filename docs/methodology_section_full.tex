\section{Methodology}
\label{sec:methodology}

This repository release is scoped to deterministic rerendering of final manuscript figures.
It is not a training-pipeline release.
The methodology here therefore describes the reproducibility contract for the final plotting package (Fig1--Fig7).

\subsection{Release Scope}
The package contains:
\begin{enumerate}
\item Seven standalone plotting scripts under \texttt{experiments/final\_scripts/}.
\item Frozen figure inputs under \texttt{outputs/final\_plots/} (NPZ/CSV snapshots).
\item Rendered final PDF/PNG artifacts under \texttt{outputs/final\_plots/}.
\item Deterministic manifest utilities in \texttt{tools/} to build and verify artifact checksums.
\end{enumerate}

Excluded from this release scope are model training/evaluation stacks and claim-level experiment pipelines.
Benchmark-side disclosure for the curated analysis reruns is documented separately in
\texttt{docs/benchmark\_reporting\_protocol.md}.

\subsection{Deterministic Rendering Protocol}
Each final plotting script is self-contained (local style + fixed default inputs) and renders a fixed figure family:
\begin{itemize}
\item Fig1: target sharpness over a $\beta$ sweep.
\item Fig2: IQP $\sigma$--$K$ recovery panel from frozen snapshot.
\item Fig3: TV/BSHS seed-mean scatter (default dual-axis boxplot at $\beta=0.90$).
\item Fig4: mechanistic visibility recovery from frozen snapshot.
\item Fig5: visible/invisible holdout recovery from frozen snapshot.
\item Fig6: $\beta$-sweep recovery grid from frozen snapshot.
\item Fig7: appendix ablation ($\beta=0.8$, fixed $n$-sweep snapshot).
\end{itemize}

Given unchanged frozen inputs and script defaults, rerendered outputs are deterministic up to platform-dependent rendering noise.

\subsection{Artifact Integrity Model}
Integrity is tracked by a manifest over \texttt{outputs/final\_plots} files.
For each file, the manifest records:
\begin{equation}
(\texttt{path},\ \texttt{size\_bytes},\ \texttt{sha256}).
\end{equation}

Verification consists of checking file existence and exact equality of size and SHA-256 digest against the manifest.
In strict mode, the verifier additionally checks that no unexpected files are present in the artifact root (excluding manifest files and \texttt{.DS\_Store}).

\subsection{Reproducibility Workflow}
The release workflow has three deterministic steps:
\begin{enumerate}
\item Rerender Fig1--Fig7 using the seven scripts in \texttt{experiments/final\_scripts/}.
\item Rebuild manifest files via \texttt{tools/build\_final\_manifest.py}.
\item Verify artifacts via \texttt{tools/verify\_final\_manifest.py --strict 1}.
\end{enumerate}

This workflow defines the publication-grade reproducibility contract for the current repository state.
